# Руководство по переносу дизайна из Lovable.dev

## Шаги для экспорта из Lovable.dev

1. **Откройте ваш проект в Lovable.dev**
   - Войдите в свой аккаунт
   - Откройте проект с нужным дизайном

2. **Экспорт кода**
   - Нажмите на кнопку "Export" или "Download" в интерфейсе Lovable
   - Или используйте меню проекта → "Export Code"
   - Скачайте ZIP архив с кодом

3. **Альтернативный способ (если экспорт недоступен)**
   - Откройте DevTools в браузере (F12)
   - Найдите код компонентов в исходном коде страницы
   - Скопируйте нужные компоненты вручную

## Что делать после экспорта

После того как вы получите код из Lovable, выполните следующие шаги:

### Вариант 1: Автоматическая интеграция (рекомендуется)

1. Распакуйте ZIP архив из Lovable
2. Запустите скрипт интеграции:
   ```bash
   npm run import:lovable <путь-к-распакованному-проекту>
   ```

### Вариант 2: Ручная интеграция

1. **Компоненты** → скопируйте в `components/`
2. **Страницы** → адаптируйте под структуру `app/`
3. **Стили** → проверьте совместимость с Tailwind конфигом
4. **Зависимости** → добавьте недостающие пакеты в `package.json`

## Адаптация кода из Lovable

### Основные отличия

1. **Структура проекта**
   - Lovable может использовать другую структуру папок
   - Адаптируйте под Next.js App Router структуру

2. **Импорты**
   - Замените относительные пути на алиасы `@/`
   - Проверьте импорты компонентов UI

3. **Стили**
   - Lovable использует TailwindCSS (совместимо)
   - Проверьте использование кастомных классов
   - Убедитесь, что цвета соответствуют вашей теме

4. **Компоненты**
   - Замените компоненты Lovable на shadcn/ui где возможно
   - Проверьте использование `"use client"` директивы

5. **Data Fetching**
   - Замените прямые `fetch` на TanStack Query хуки
   - Используйте API из `lib/api/`

## Чеклист после переноса

- [ ] Все компоненты используют правильные импорты (`@/`)
- [ ] Добавлены директивы `"use client"` где необходимо
- [ ] Стили совместимы с текущей темой
- [ ] Все зависимости установлены
- [ ] Код проходит линтинг (`npm run lint`)
- [ ] TypeScript компилируется без ошибок
- [ ] Все тексты в UI на русском языке
- [ ] Data fetching через TanStack Query
- [ ] API вызовы через `lib/api.ts`

## Полезные команды

```bash
# Проверка линтинга
npm run lint

# Автоисправление линтинга
npm run lint:fix

# Форматирование кода
npm run format

# Проверка типов
npx tsc --noEmit
```
